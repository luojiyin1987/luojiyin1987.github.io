编译安装ImageMagick

二 21 六月 2016
By luojiyin

由于CentOS 6.7通过yum方式按照的ImageMagick版本是6.7。但是最新版本 6.9.3-7。在压缩相同源文件，出来的相同尺寸的图片的文件小50%以上，效果明显，值得自己的编译源代码安装。通过google 找到解决办法。google 人类的希望。 参考教程链接

具体过程

删除系统原来的ImageMagick

yum remove ImageMagick

准备好编译环境

yum -y groupinstall 'Development Tools'yum -y groupinstall 'Development Tools'

yum -y install bzip2-devel freetype-devel libjpeg-devel libpng-devel libtiff-devel giflib-devel zlib-devel ghostscript-devel djvulibre-devel libwmf-devel jasper-devel libtool-ltdl-devel libX11-devel libXext-devel libXt-devel lcms-devel libxml2-devel librsvg2-devel OpenEXR-devel yum -y install bzip2-devel freetype-devel libjpeg-devel libpng-devel libtiff-devel giflib-devel zlib-devel ghostscript-devel djvulibre-devel libwmf-devel jasper-devel libtool-ltdl-devel libX11-devel libXext-devel libXt-devel lcms-devel libxml2-devel librsvg2-devel OpenEXR-devel

因为我的系统安装了php，没像教程中加入了 php-devel.否则yum检测依赖关系时报错,第二条命令执行失败.教程作者好积极推广PHP

下载ImageMagick源代码 解压

wget http://www.imagemagick.org/download/ImageMagick.tar.gz

tar xvzf ImageMagick.tar.gz

cd ImageMagick*

编译安装

./configure

make

make install

如果是多核CPU的,make可以加参数,像四核 make -j 4, 二核是 make -j 2, 类推. 没事时,多看man,有收获.

版本查看

convert --version

对比原版本号 确认一下

验证

convert x.jpg x.png

查看图片是否正常转换.
